# Folder Structures Reference

These structures are recommended for consistent organization across different domains.

## 1. Project Structure (Software/Tech)
```
[Project_Name]/
├── 01_Strategy/            # Business logic, requirements, research
├── 02_Design/              # UI/UX, mockups, brand assets
├── 03_Development/         # Source code (git root)
├── 04_Documentation/       # API docs, user guides, setup instructions
├── 05_Test_QA/             # Test cases, logs, bug reports
├── 06_Deployment/          # Infra code, scripts, configs
└── INDEX.md
```

## 2. Company Structure (General)
```
[Company_Name]/
├── 00_BOD/                 # Board of Directors, high-level strategy
├── 01_HR_Admin/            # Legal, HR, Office management
├── 02_Finance_Accounting/  # Billing, taxes, payroll
├── 03_Sales_Marketing/     # Campaigns, leads, CRM
├── 04_Operations/          # Production, logistics
├── 05_Tech_Product/        # Product design, R&D
├── 98_Templates/           # Company-wide document templates
└── INDEX.md
```

## 3. Personal Organization
```
[Name_Folder]/
├── 01_Work/                # Professional projects and tasks
├── 02_Learning/            # Courses, books, certificates
├── 03_Finance/             # Personal budget, taxes, bank
├── 04_Personal/            # Health, travel, hobbies
├── 05_Archive/             # Completed items, old photos
└── INDEX.md
```

## 4. Generic Media/Assets
```
[Assets]/
├── 01_Images/
│   ├── Raw/
│   └── Final/
├── 02_Videos/
├── 03_Audio/
├── 04_3D_Models/
└── 05_Fonts/
```
